# Desktop App Details

## Installed Files

`./scripts/install-user-app.sh` installs Linux Agent Island into the current user account:

```text
~/.local/share/linux-agent-island/venv
~/.local/bin/linux-agent-island
~/.local/bin/lai
~/.config/systemd/user/linux-agent-island.service
~/.local/share/applications/linux-agent-island.desktop
~/.local/share/icons/hicolor/scalable/apps/linux-agent-island.svg
```

Runtime files stay under:

```text
~/.local/state/linux-agent-island/
```

Important runtime files:

```text
sessions.json
logs/backend.log
logs/frontend.log
logs/hotkeys.log
logs/tray.log
logs/daemon.log
events.sock
```

User settings live at:

```text
~/.config/linux-agent-island/settings.json
```

## CLI and Service

The installed CLI is generated by Python packaging from:

```toml
[project.scripts]
linux-agent-island = "linux_agent_island.cli:main"
lai = "linux_agent_island.cli:main"
```

The service runs:

```text
linux-agent-island daemon
```

The desktop launcher runs:

```text
linux-agent-island open
```

This keeps the launcher from starting a second daemon when the service is already running.

Available commands:

```bash
lai open
lai toggle
lai settings
lai status
linux-agent-island daemon
linux-agent-island open
linux-agent-island toggle
linux-agent-island settings
linux-agent-island status
linux-agent-island highlight-selected
linux-agent-island codex login --label Work
linux-agent-island codex status
linux-agent-island codex usage
linux-agent-island codex usage <account>
linux-agent-island codex sync-auth <account>
linux-agent-island codex accounts list
linux-agent-island codex accounts switch <account>
linux-agent-island codex accounts rename <account-id> <label>
linux-agent-island codex accounts set-default <account-id>
linux-agent-island codex accounts delete <account-id>
linux-agent-island codex accounts import-current
linux-agent-island install-hooks
linux-agent-island uninstall-hooks
```

`lai` is a short alias for the main CLI entry point, so all subcommands are available through either command name. The Codex account login button in the settings window does not own a separate login implementation anymore. It calls the backend D-Bus `StartCodexDeviceLogin` method, and the backend launches regular `linux-agent-island codex login` asynchronously. The CLI owns the full login lifecycle, writes shared login-state metadata under `~/.config/linux-agent-island/codex-accounts/`, and the UI refreshes when the CLI exits. Keeping Codex actions under a dedicated `codex` command group leaves room for future subcommands without bloating the top-level CLI. The legacy `linux-agent-island codex-login` alias still works, but now warns that it is deprecated.

Keyboard shortcuts:

- Global `Ctrl+Alt+I`: focus the island; if the island is already active, minimize it and restore the previous X11 window.
- Window-local `Ctrl+H`: toggle highlight on the selected session row.

Default labels for imported Codex accounts prefer the locally decoded token email in this order: `id_token.email`, `access_token.email`, `access_token["https://api.openai.com/profile"].email`, then fallback names.

## Settings

Supported settings:

```json
{
  "top_bar_gap": 8,
  "log_level": "INFO",
  "start_on_login": true
}
```

`top_bar_gap` applies when the island is repositioned. `log_level` applies after service restart. `start_on_login` maps to `systemctl --user enable/disable linux-agent-island.service`.

## Hooks

The backend installs or refreshes Claude Code and Codex CLI hooks at startup.

Managed files:

```text
~/.claude/settings.json
~/.codex/hooks.json
```

Installed hook commands use the venv package module:

```bash
~/.local/share/linux-agent-island/venv/bin/python -m linux_agent_island.hooks claude <event>
~/.local/share/linux-agent-island/venv/bin/python -m linux_agent_island.hooks codex <event>
```

Claude managed events:

```text
UserPromptSubmit
PreToolUse
PostToolUse
PermissionRequest
Notification
Stop
SessionStart
SessionEnd
PreCompact
```

Codex managed events:

```text
SessionStart
UserPromptSubmit
Stop
```

Old Linux Agent Island-managed source-path hooks and copied Codex shims are pruned automatically. User-defined hooks are preserved.

## Development Mode

`scripts/run-dev.sh` starts backend and frontend from the checkout and sets:

```bash
LINUX_AGENT_ISLAND_HOOK_COMMAND_PREFIX="PYTHONPATH=<repo> /usr/bin/python3 -m linux_agent_island.hooks"
```

That makes hook events use the working tree during development. Restore installed hooks afterward with:

```bash
linux-agent-island install-hooks
```

## Uninstall Notes

There is not a dedicated uninstall script yet. Manual removal:

```bash
systemctl --user stop linux-agent-island.service
systemctl --user disable linux-agent-island.service
rm -f ~/.config/systemd/user/linux-agent-island.service
rm -f ~/.local/share/applications/linux-agent-island.desktop
rm -f ~/.local/share/icons/hicolor/scalable/apps/linux-agent-island.svg
rm -f ~/.local/bin/linux-agent-island
rm -f ~/.local/bin/lai
rm -rf ~/.local/share/linux-agent-island
systemctl --user daemon-reload
```

To remove only managed Claude/Codex hooks before deleting the app:

```bash
linux-agent-island uninstall-hooks
```
